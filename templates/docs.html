Quick Guide

Terminal

Type any Linux/Termux command in the input field. Press Ctrl+Enter or tap [EXEC] to run. Output streams in real-time. The terminal supports cd natively to change working directory, so you can navigate the filesystem without leaving the web UI. If a process hangs for 10 seconds, a warning appears with a KILL button to terminate it. Each command entry shows timestamp, exit code badge, and action buttons to copy output or delete the entry. The terminal keeps up to 30 entries visible and up to 50 commands in history.

  • ↑ / ↓ — Navigate command history
  • Ctrl+Enter — Execute command
  • Start typing to see suggestions from previous commands
  • Hung process auto-detection after 10s with kill option

File Manager

Click [FILES] in the header to open the file manager dialog. Browse directories by clicking folders, click any file to open it in the built-in editor. The editor supports line numbers, auto-indentation (including Python-style after :), Tab-to-spaces (4 spaces), and Ctrl+S to save. Use [PREVIEW] to see syntax-highlighted code via highlight.js. The toolbar provides upload from Android gallery/native file manager, refresh, and global search. Navigate up with the .. parent directory entry.

  • [SAVE] — Save file (or Ctrl+S in editor)
  • [+ NEW FILE] — Create empty file in current directory
  • [RENAME] — Rename currently open file
  • [UPLOAD] — Upload file from device
  • ↓ button on file — Download file to device
  • × button on file — Delete file/directory
  • [PREVIEW] — Syntax-highlighted read-only view

Global Search

In File Manager, click SEARCH to open the "Find in Files" dialog. Type a query and press Enter or click Search. Results are grouped by file with match counts. Click any match to jump directly to that line in the editor with the matching text highlighted and selected. Toggle case-sensitive mode with the Aa button. The search uses grep on the server across Python, JS, TS, HTML, CSS, JSON, Markdown, YAML, TOML, shell, and config files, excluding node_modules, .git, __pycache__, and dist directories. Results are capped at 300 lines.

Git / GitHub Panel

Click [GIT] in the header to open the Git panel. This is a full-featured Git client built directly into the web UI. It provides 6 tabs covering the entire Git workflow — from initializing a repository to pushing commits to GitHub. The panel displays the current branch, sync status (ahead/behind count), and comprehensive file change tracking with staged, unstaged, and untracked file groups. Each file shows its change type and provides stage, unstage, and discard actions.

  • Status Tab — View staged, modified, untracked files. Stage/unstage/discard individual files or in bulk.
  • Commit Tab — Write commit messages, stage all, and commit. Supports Ctrl+Enter to commit.
  • Sync Tab — Push, pull, and fetch from remote. Auto-detects missing upstream and offers to set it.
  • Log Tab — View commit history with hash, message, author, and relative time (up to 50 commits).
  • Branches Tab — List all branches (local + remote), checkout, or create new branches.
  • Config Tab — Set global Git identity (name + email) and manage remote URLs. For GitHub, embed your Personal Access Token in the URL: https://TOKEN@github.com/user/repo.git

Status Bar

The status bar at the bottom of the terminal displays real-time system information. It polls the server to detect connection status and updates the current working directory live every 2 seconds. The command counter increments with each executed command, and the uptime timer starts when the page loads. A green dot means connected, red means offline.

  • Left — Connection status dot + label (CONNECTED / OFFLINE)
  • Center — Live current working directory (updates every 2s)
  • Right — Command count + session uptime timer

PWA Install

Yuyutermux is a Progressive Web App. Install it as a standalone Android app — no browser address bar, icon on home screen, runs like a native app. An automatic install prompt appears when conditions are met. Alternatively, use "Add to Home Screen" from the browser menu. The Service Worker caches HTML pages and CSS for offline access. API requests are never cached to ensure fresh data. Dynamic assets (JS modules, CDN fonts) are cached at runtime on first load. To force a cache refresh, use the cache-bust feature in run.sh or clear browser data.

CLI: run.sh (Server Manager)

The run.sh script is an interactive terminal menu for managing the Yuyutermux server. It uses Waitress as the production WSGI server (not Flask's built-in dev server). The script handles port conflict detection, PID file management, graceful shutdown with SIGTERM followed by SIGKILL fallback, and displays the server's local IP for cross-device access.

  • Start — Launch Waitress on port 5000, auto-detect and free busy ports
  • Stop — Graceful shutdown with PID tracking
  • Restart — Stop + Start in sequence
  • Status — Show PID, uptime, port status, log size, SW cache version
  • Log — View last 20 lines of server log
  • Bust Cache — Increment Service Worker cache version (no restart needed)

CLI: edit.sh (File Wizard)

A standalone CLI tool for quick file editing directly from Termux terminal. It scans the project directory for code files (max depth 3), lets you pick one interactively (supports fzf for fuzzy search), then choose an action: edit in nano, delete, empty and edit, or copy to clipboard. Optional dependencies: fzf for fuzzy file picker, termux-clipboard-set for clipboard support.

Security Model

Yuyutermux is designed for single-user Termux environments. All path access is validated through validate_path() which resolves the real path and ensures it stays within the project directory (~/Yuyutermux). This prevents directory traversal attacks even if symlinks are involved. Commands are parsed with shlex.split() to prevent shell injection, and file uploads are restricted to the project directory. The cd command is handled natively in Python rather than through a shell, further reducing attack surface.

  • Path Traversal Guard — All file operations use validate_path() which resolves symlinks and checks against PROJECT_DIR
  • Command Injection Prevention — Commands parsed via shlex.split(), cd handled in Python
  • File Size Limit — Read operations capped at 1MB to prevent memory exhaustion
  • Upload Restriction — Upload paths validated against project directory
  • Search Scope — Grep search limited to project directory with exclude patterns
  • API Isolation — Each route is a Flask Blueprint with scoped URL prefixes

JavaScript Modules

The frontend uses vanilla ES modules with zero build tools. Each module exports a single object or function. The entry point is app.js which imports and initializes all modules on DOMContentLoaded.

app.js
Entry point. Initializes splash screen, input, terminal, file manager, Git, PWA, and all utility modules.
terminal.js
Core: Terminal (streaming exec, history, kill), Toast (notifications), StatusBar (connection, cwd, timer), Suggestions (autocomplete dropdown).
file-manager.js
File browser, editor integration, upload/download, rename, create, preview with highlight.js, open-file-at-line for search results.
editor.js
Code editor: line numbers, scroll sync, Tab-to-spaces, auto-indent (Python : aware), Ctrl+S save, cursor insertion.
github.js
Full Git client: 6 tabs (status, commit, sync, log, branches, config), delegated event handling, upstream auto-set on push failure.
global-search.js
Find-in-files dialog with case toggle, file-grouped results, click-to-open at line, match highlighting via search-utils.js.
api.js
HTTP helper (api.get/post), HTML entity escaper, ANSI color parser, file type icon mapping for 18 extensions.
extra-keys.js
Configurable symbol/action bar. 12 default keys: 9 insert characters + 3 actions (kill, clear, history nav).
storage.js
Persists terminal entries to localStorage. Saves last 30 entries, rebinds copy/delete events on load.
modal-keyboard.js
Detects Android soft keyboard open/close on editor textarea, adds CSS class for layout adjustment.
viewport-fix.js
Sets --vvh CSS variable from visualViewport height. Handles resize and virtual keyboard changes.
search-utils.js
Regex-safe highlight pattern builder and HTML-safe match highlighting with <mark> tags.
