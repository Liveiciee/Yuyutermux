Quick Guide

Terminal

Type any Linux/Termux command in the input field. Press Ctrl+Enter or tap the send button to execute. Output streams in real-time via chunked transfer encoding (not SSE). The terminal handles cd natively in Python, so the working directory persists across commands without spawning a subshell. A hung process warning appears after 10 seconds of no output, with a KILL button that terminates the entire process group via os.killpg(). Each command entry shows a timestamp, exit code badge (green for 0, red for non-zero), and action buttons to copy output or delete the entry. The terminal keeps up to 30 entries visible on screen (older entries auto-collapse) and stores up to 50 commands in history. Commands have a maximum length of 10,000 characters and a process timeout of 5 minutes (300 seconds).

  • ↑ / ↓ — Navigate command history
  • Ctrl+Enter — Execute command
  • Start typing to see live suggestions from previous commands (top 5 matches)
  • Hung process auto-detection after 10s with kill button
  • Click any terminal entry to expand/collapse it
  • Staggered fade-out animation when clearing all entries

Extra Keys

Symbol bar above the input provides quick access to characters that are tedious to type on mobile keyboards. Each key either inserts a character at the cursor position or triggers an action. Accent-colored keys are action buttons. Every press triggers a ripple animation for tactile feedback.

File Manager

Click [FILES] in the header to open the file manager dialog. Browse directories by clicking folders, click any file to open it in the built-in code editor. The editor features real-time syntax highlighting via an overlay technique — a <pre> layer behind the textarea displays highlight.js-colored code, while the textarea sits on top with transparent text so the user sees colors through it. This means syntax highlighting is live, not a separate preview mode. The editor supports line numbers with scroll sync, auto-indentation for multiple languages (Python :, JavaScript/CSS/JSON { ( [), Tab-to-spaces (4 spaces), and Ctrl+S to save. Upload uses a <label for="uploadInput"> pattern instead of programmatic .click() to ensure file picker works on all mobile browsers. Duplicate filenames are handled with timestamp-based renaming.

  • [SAVE] — Save file (or Ctrl+S in editor)
  • [+ NEW FILE] — Create empty file in current directory
  • [RENAME] — Rename currently open file (read-write-delete sequence)
  • UPLOAD — Upload file from device (max 50MB, duplicate-safe)
  • ↓ button on file — Download file to device
  • × button on file — Delete file or directory
  • Click any file to open in editor with live syntax highlighting
  • Click .. to navigate to parent directory

Code Editor

The editor uses a dual-layer technique: a <pre><code> highlight layer sits behind the textarea. When highlighting is active, the textarea's text color becomes transparent and its background becomes transparent, revealing the colored syntax layer beneath. The caret stays visible via caret-color. This approach provides real-time syntax highlighting without a separate preview mode. Highlighting is debounced at 100ms to avoid performance issues during rapid typing. The editor uses document.execCommand('insertText') for Tab and Enter key handling to preserve browser undo/redo history.

  • Languages — Python, JavaScript, TypeScript, HTML, CSS, JSON, Markdown, YAML, TOML, INI
  • Auto-indent — After : (Python), { (JS/JSON), (, [ — adds 4-space indent
  • Undo/Redo — Preserved via execCommand insertText, not .value assignment
  • Line Numbers — Optimized for large files (chunked rendering for 1000+ lines)
  • Scroll Sync — Gutter, textarea, and highlight layer scroll in unison
  • Viewport Aware — Adapts to virtual keyboard height via visualViewport API

Global Search

In File Manager, click SEARCH to open the "Find in Files" dialog. Type a query and press Enter or click Search. Results are grouped by file with match counts per file, and each result is expandable/collapsible. Click any match to jump directly to that line in the editor with the matching text highlighted and selected, plus a brief orange flash on the editor wrapper. Toggle case-sensitive mode with the Aa button. The search uses grep on the server across 13 file types (Python, JS, TS, HTML, CSS, JSON, Markdown, TXT, Shell, YAML, YML, TOML, CFG), excluding node_modules, .git, __pycache__, and dist directories. Results are capped at 300 lines and each match line is truncated at 500 characters. The search query is limited to 200 characters.

Git / GitHub Panel

Click [GIT] in the header to open the Git panel. This is a full-featured Git client with 6 tabs covering the entire Git workflow. The panel displays the current branch name, sync status (ahead/behind count), and comprehensive file change tracking. Remote URLs containing tokens are automatically masked in the status display (shown as https://***@github.com/...). Push operations auto-detect missing upstream and offer to set it with --set-upstream. Branch names and remote names are validated against regex patterns to prevent injection.

  • Status Tab — View staged, modified, untracked files with change type badges. Stage/unstage/discard individual files or in bulk (UNSTAGE ALL, STAGE ALL, ADD ALL).
  • Commit Tab — Write commit messages (max 5000 chars), stage all, and commit. Supports Ctrl+Enter to commit.
  • Sync Tab — Push (with force option), pull, and fetch from remote. Auto-detects missing upstream and offers to set it.
  • Log Tab — View commit history with short hash, message, author, and relative time (up to 50 commits).
  • Branches Tab — List all branches (local + remote) with current marker. Checkout existing branches or create new ones.
  • Config Tab — Set global Git identity (name + email, max 200 chars each, control characters rejected) and manage remote URLs. For GitHub, embed your Personal Access Token: https://TOKEN@github.com/user/repo.git

Status Bar

The status bar at the bottom of the terminal displays real-time system information. Connection status is checked via /api/health (lightweight, no directory scanning) every 15 seconds. The working directory is polled every 5 seconds. The command counter increments with each executed command, and the uptime timer starts when the page loads and updates every second. A green pulsing dot means connected, a red pulsing dot means offline. On mobile (<640px), the center CWD display is hidden to save space.

  • Left — Connection status dot + label (CONNECTED / OFFLINE)
  • Center — Live current working directory with ~/ shorthand (updates every 5s, hidden on mobile)
  • Right — Command count + session uptime timer (MM:SS)

PWA Install

Yuyutermux is a Progressive Web App. Install it as a standalone Android app — no browser address bar, icon on home screen, runs like a native app in standalone mode. An automatic install prompt appears when conditions are met, or use "Add to Home Screen" from the browser menu. The Service Worker pre-caches HTML pages and CSS for dynamic assets (JS modules, CDN fonts) at runtime on first load, and never caches API requests. Cache is limited to 100 entries to prevent flooding. Internal URLs (file://, data:text/html) are blocked for security. To force a cache refresh, use the "Bust Cache" feature in run.sh which increments the cache version (currently v16) without requiring a server restart.

Authentication

On first visit, the browser redirects to /login where the auth token is entered manually. The token is generated fresh in-memory on every server start via secrets.token_urlsafe(32), or can be set via the YUYUTERMUX_TOKEN environment variable. On successful login, two cookies are set (both valid 30 days, SameSite=Lax):

  • yuyu_token — HttpOnly cookie storing the actual auth token. Not readable by JavaScript. Used by the backend for server-side auth (auto-sent by the browser on every request).
  • yuyu_authed — Non-HttpOnly marker cookie with value 1. Readable by JavaScript via document.cookie.includes('yuyu_authed'). Used by the frontend to show/hide UI elements like the logout button.

Backend auth (check_auth()) checks the Authorization: Bearer <token> header first, then falls back to the yuyu_token cookie. Both use secrets.compare_digest() for constant-time comparison. Logout clears both cookies and redirects to /login. Error messages are uniform — they never reveal whether the token was empty, wrong, or the system has no token configured.

  • Token displayed once in server log on startup
  • View token via run.sh → Show Token, or set via YUYUTERMUX_TOKEN env var
  • yuyu_token is HttpOnly (prevents XSS theft); yuyu_authed is non-HttpOnly (needed for frontend UI state)
  • All API routes require auth except /api/health, /api/auth/login, /api/auth/logout, /api/verify-token

CLI: run.sh (Server Manager)

The run.sh script is an interactive terminal menu for managing the Yuyutermux server. It uses Waitress as the production WSGI server (not Flask's built-in dev server). Requires waitress-serve to be installed. The script handles port conflict detection using ss (preferred) or netstat (fallback), PID file management, graceful shutdown with SIGTERM followed by SIGKILL fallback, and displays the server's local IP for cross-device access.

  • Start — Launch Waitress on 0.0.0.0:5000, auto-detect and free busy ports. Displays auth token on startup.
  • Stop — Graceful shutdown with PID tracking (SIGTERM, then SIGKILL after 5s)
  • Restart — Stop + Start in sequence
  • Status — Show PID, uptime, port status, log size, SW cache version, auth token
  • Log — View last 20 lines of server log
  • Bust Cache — Increment Service Worker cache version (no restart needed)
  • Show Token — Display current auth token from env var or server log

CLI: edit.sh (File Wizard)

A standalone CLI tool for quick file editing directly from Termux terminal. It scans the project directory for code files up to depth 3 (Python, JS, TS, HTML, CSS, JSON, TXT, MD, YAML, TOML, SH, CFG), lets you pick one interactively — with fzf for fuzzy search if available, or numbered list otherwise — then choose an action: edit in nano, delete, empty and edit, or copy to clipboard. Optional dependencies: fzf for fuzzy file picker with preview, termux-clipboard-set for clipboard support (with 1MB size warning).

Security Model

Yuyutermux is designed for single-user Termux environments. The security model operates on multiple layers: authentication, input validation, output sanitization, and process isolation. The server binds to 0.0.0.0 via Waitress (configured in run.sh) but Flask defaults to 127.0.0.1 for direct launches. Debug mode is off by default. All error messages sanitize real filesystem paths, replacing $HOME with ~ and ~/Yuyutermux for the project directory.

  • Token Auth — In-memory token via secrets.token_urlsafe(32), fresh each server start. Or set via YUYUTERMUX_TOKEN env var. Constant-time comparison via secrets.compare_digest(). Two-cookie system: yuyu_token (HttpOnly, SameSite=Lax) for backend auth, yuyu_authed (non-HttpOnly, SameSite=Lax) for frontend UI state.
  • Login Page — /login auto-redirect if unauthenticated. Uniform error messages (never reveals if token is empty vs wrong). Spinner state during login attempt.
  • Security Headers — X-Content-Type-Options: nosniff, X-Frame-Options: DENY, X-XSS-Protection: 1; mode=block, Referrer-Policy: no-referrer, Permissions-Policy: camera=(), microphone=(), geolocation=(), Cache-Control: no-store, no-cache, Pragma: no-cache
  • CSP + SRI — Content Security Policy in index.html (also in login.html). Subresource Integrity hashes on all CDN assets (highlight.js JS + CSS, Google Fonts).
  • Rate Limiting — In-memory per-IP with periodic cleanup (every 2 min). Default 30 req/60s. Stricter 15 req/60s on command execution. 10 req/60s on file upload.
  • Blocked Commands — 77 commands blocked: rm, sudo, curl, wget, python, node, env, exec, eval, source, busybox, xargs, nohup, find, awk, sed, and more. Includes indirection vectors, shell interpreters, and crypto tools.
  • Path Traversal Guard — All file operations use validate_path() which rejects .., resolves symlinks via os.path.realpath(), and checks against PROJECT_DIR. Separate validate_path_terminal() for cd (restricted to home dir).
  • Command Injection Prevention — Commands parsed via shlex.split(); cd handled natively in Python; no shell=True in file routes; search uses list args for grep.
  • Process Isolation — Terminal processes created with os.setsid() for process group isolation. Killed via os.killpg() to prevent orphans. 5-minute max timeout per process.
  • CSRF Mitigation — POST/PUT/DELETE require application/json or multipart/form-data Content-Type (415 rejected otherwise). Login/logout exempted.
  • Protected Files — Write blocks: .auth_token, .env, .htaccess, .htpasswd, .bashrc, .bash_profile, .profile, .bash_history, passwd, shadow. Also blocks writes to .ssh/ paths. Delete protects (project root only): app.py, utils.py, run.sh, .auth_token.
  • Output Sanitization — All API error messages sanitize filesystem paths. Git remote URLs mask tokens. Git commit messages/authors sanitized. Rate limit store auto-cleaned.
  • Stored XSS Prevention — localStorage history sanitized on load: removes <script>, inline event handlers (onclick, etc.), and javascript: URLs. Version-based invalidation discards old data.

Frontend Architecture

app.js

Entry point. Initializes all modules with try-catch isolation (one failure doesn't kill others). Manages interval cleanup on page unload. Binds global keyboard shortcuts (Ctrl+K, Ctrl+L, Ctrl+F, Ctrl+G, Ctrl+/). Controls splash screen animation sequence.

terminal.js

Terminal core (Terminal), toast notifications (Toast), status bar (StatusBar), command suggestions (Suggestions). Handles streaming output, entry animations, virtual scroll collapse, command history (50 entries), and process kill. Lazy DOM reference to avoid null crashes.

editor.js

Code editor module (Editor). Dual-layer syntax highlighting with highlight.js overlay. Auto-indent for Python/JS/CSS/JSON/HTML. Line numbers with optimized rendering for large files. Viewport-aware with virtualKeyboard API. Memory-safe with destroy() cleanup.

file-manager.js

File browser and operations (FileManager). Directory listing, file open/read/save/delete/rename/create. Upload with label-based file picker for mobile compatibility. openFileWithLine() for global search integration with line highlight flash.

github.js

Git panel (GitHub). 6-tab interface: Status, Commit, Sync, Log, Branches, Config. Delegated event binding for dynamic content. Auto-upstream detection on push failure. Token-masked remote display.

api.js

HTTP client (api), auth management (Auth), HTML escaping (esc), ANSI color parser, file type icons. Backend auth uses HttpOnly yuyu_token cookie (auto-sent by browser, not readable by JS). Frontend UI state uses non-HttpOnly yuyu_authed marker cookie. Auto-redirect to /login on 401.

storage.js

Terminal history persistence (Storage). Saves last 30 entries to localStorage. Version-based invalidation (v2). XSS sanitization strips scripts, event handlers, and javascript: URLs on restore. Rebinds event handlers after DOM restore.

global-search.js

Find-in-files dialog (GlobalSearch). Dynamic dialog creation. Server-side grep search with case toggle. Click-to-open with line scroll and selection. Expandable/collapsible file groups with match counts.

extra-keys.js

Symbol bar (ExtraKeys). 12 keys: 9 character inserts + 3 actions (kill, clear, history nav). Ripple animation on press. Inserts at cursor position.

modal-keyboard.js

Virtual keyboard detection (ModalKeyboard). Adds CSS class to modal when editor is focused, collapsing the file list to give more space to the textarea on mobile.

search-utils.js

Search utilities. Provides highlightMatch() for keyword highlighting in search results and escapeHtml() for safe HTML rendering. Regex pattern builder with escaping.

Caveats & Known Limitations

No Interactive / PTY Terminal

Yuyutermux streams command output but does NOT provide a real pseudo-terminal (PTY). There is no stdin — you cannot interact with a running process after it starts. Commands that expect interactive input (e.g. python REPL, ftp, password prompts from ssh, vim) will hang until the 5-minute timeout kills them. Use the KILL button or CTRL+C key in the extra keys bar to terminate a hung process.

Command Blocklist is Not a Sandbox

The blocked command list (rm, sudo, curl, bash, python, etc.) prevents the most obvious misuse but is NOT a real security sandbox. A determined attacker with valid token access can still do damage via allowed commands. Yuyutermux is intended for trusted single-user use on a local device, not as a public-facing service.

File Manager Scope: Project Directory Only

The file manager ([FILES]) is locked to ~/Yuyutermux and cannot browse outside it. The terminal's cd command allows navigation within the entire home directory (~), but all file read/write/delete/upload operations are restricted to the project directory. Editing files outside ~/Yuyutermux requires using the terminal directly.

Auth Token Resets on Every Server Restart

If no YUYUTERMUX_TOKEN environment variable is set, a fresh random token is generated each time the server starts. Existing browser sessions with the old cookie will be logged out on the next API call. To persist the token across restarts, set the env var in your shell config (e.g. ~/.bashrc) or use the YUYUTERMUX_TOKEN var when starting via run.sh.

File Size Limits

Files larger than 1 MB cannot be opened or edited in the browser editor (the API rejects them). Upload is capped at 50 MB per file. The file write API accepts content up to 5 MB. There is no in-browser progress indicator for large uploads — the UI will appear frozen until the upload completes or fails.

Syntax Highlighting Requires CDN

The code editor's syntax highlighting depends on highlight.js 11.9 loaded from the Cloudflare CDN. If the device is offline or the CDN is unreachable, the editor still works but shows plain unstyled text. The editor detects this gracefully — it removes the highlighting-on class and falls back to normal textarea rendering.

Rate Limiting is In-Memory and Per-Restart

Rate limit counters are stored in Python memory, not persisted to disk. They reset completely on server restart. Limits are per-IP per-endpoint: 30 req/60s general, 15 req/60s for command execution, 10 req/60s for uploads, 5 req/60s for login attempts. On localhost, all requests appear to come from 127.0.0.1, so rate limits are effectively global when accessed locally.

Service Worker Caches Aggressively

The Service Worker caches HTML pages and CSS on install, and JS modules at runtime. After updating project files, the browser may serve stale cached versions. Use Bust Cache in run.sh to increment the SW cache version and force a fresh fetch. API calls (/api/*) are never cached. Hard-reload in Brave (⋮ → Reload) bypasses the SW for one request only.

Single-User Only

Yuyutermux has no concept of multiple users, sessions, or workspaces. There is one auth token, one working directory state, and one set of active processes server-wide. Running multiple browser tabs simultaneously will share the same terminal CWD state, and killing a process in one tab affects all tabs.

Git Operations Require a Valid git Install

All Git panel features call the system git binary via subprocess. If git is not installed in Termux (pkg install git), all Git API calls will return "git not found" errors. GitHub push/pull over HTTPS requires embedding a Personal Access Token (PAT) in the remote URL — SSH key auth is not configurable through the UI.

No Undo for File Delete or Git Discard

Deleting a file (including directories, via shutil.rmtree) and discarding Git changes (git restore) are permanent and irreversible from within the app. There is a confirmation dialog for both operations, but no recycle bin or undo history. Make sure you have backups or are on a git-tracked project before deleting.
